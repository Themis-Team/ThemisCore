Detailed Description

To facilitate model-data comparisons, here a fixed interface that may be assumed by likelihoods, etc., for generating model flux data is defined. All models that produce flux data should be child classes of model_flux, thereby inheriting this interface. The key function is flux which takes a datum_flux, providing access to all of the relevant accoutrements of a data point, and an accuracy parameter, and returns the model flux.

This class contains multiple purely virtual functions, making it impossible to generate an explicit instantiation.
